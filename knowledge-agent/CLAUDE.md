# knowledge-agent

## 프로젝트 목적 (가장 중요)

이 프로젝트의 최종 목적은 **측정 → 학습 추천/순서 결정**이다.

AI 논문을 파이프라인으로 먹여 **지식맵**(노드=개념+정의, 엣지=`is_a`/`part_of`)을 누적하고,
그 맵으로 사용자 본인이 무엇을 알고/모르는지 **커버리지를 측정**한 뒤, 거기서 멈추지 않고
**무엇을 어떤 순서로 학습할지 추천·결정**하는 것까지가 목적이다.
→ 단순 지식그래프 구축이 아니다. "맵은 내 지식 커버리지를 재는 측정 도구"다.

**함의:** 노드 `definition.text`가 틀리면 그 틀린 정의로 커버리지를 재고 학습 추천까지
어긋난다. 그래서 **정의 오염(개념과 어긋난 정의)은 표기 중복보다 위험**하다.
노드의 `mastery`/`last_probed`/`_seed_intent`는 이 학습 루프용 필드다.

## 설계 철학 (`docs/PIPELINE_POLICY.md` 참조)

- **precision-first**: 틀린 것보다 빈 게 낫다. 충돌·저신뢰 정의는 운영맵(`data/knowledge_map.json`)에 안 박는다.
- **enrichment**: 빈약·오염 정의는 버그가 아니라 "그 개념의 본진 논문이 아직 안 옴". 논문이 쌓이며 채워진다.
- **정보 안 버리고 판단 가능한 곳에서 결정**: 정의를 다중 보관하고, 하류 단계가 거리·깨끗함으로 best를 고른다.
- **실측 전 가정 금지**: 추측으로 단정하지 말고 산출물에서 증거를 찾는다.

## 파이프라인 (코드: `agents/`, 산출물: `data/parsed/{paper_id}/`)

```
01 수집 → 02 파싱 → 03 요약 → 04 개념·정의 추출 → 04b 표기정규화
→ 05 정렬(new/merge) → 06 관계 → 07 CRA(모순감지) → 08 통합(SELECT, 맵 반영)
```

- 환경: uv, Python 3.12, gpt-4o-mini. 실행 예: `uv run python -m agents.agent_04_entity_extraction --paper <id> --run`
- 맵은 의도적으로 비운 뒤 ToT 논문(2305.10601) 한 편만 태운 상태일 수 있다.
