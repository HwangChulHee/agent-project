# 파이프라인 설계 정책 (world-model 구축)

> 이 문서는 **"왜 이렇게 정했나"**를 기록한다. "무엇을 했나"는 코드를 보면 되지만,
> "왜"는 코드에 안 남으므로 여기 박아둔다. 다음 세션 복귀 + 면접 서사용.
>
> 기준 시점: ReAct 논문(2210.03629) 한 편을 IA → align까지 통과시킨 직후.
> (참고: `docs/KARMA_AGENTS.md`는 구버전 설계 기준이라 현재와 불일치 — 갱신/폐기 대상)

---

## 0. 프로젝트 전체 그림

개인 지식 시스템, 3단계. 지금은 ①을 구축 중.

```
① world-model 구축   논문들을 읽어 "분야 지식 지도"를 자동 생성   ← 현재
② self-model         그 지도에서 내가 뭘 아는지(mastery) 측정
③ 추천               커버 안 된 곳 기반 학습 추천
```

①은 KARMA(논문→지식그래프 자동구축)의 파이프라인을 참고하되, 우리 목적
("정확한 분야 그래프 완성"이 아니라 **"내 커버리지 지형도"**)에 맞게 가지치기·변형한다.

핵심 철학: **맵은 지형도지 디테일 저장소가 아니다.** 내가 아는 영역/모르는 영역을
표시하는 게 목적이고, 디테일하게 파고드는 건 그다음(self-model 카드 → 프론티어 LLM) 몫.
이 한 가지가 아래 거의 모든 결정을 좌우한다.

---

## 1. 파이프라인 흐름

```
PDF
 │
 ▼ 01 IA   (Ingestion)         PDF 파싱(pymupdf4llm) + 청소(references·표·그림 제거)
 ▼ 02 RA   (Reader)            ## 헤더로 섹션 분할 (관련성 컷은 보류)
 ▼ 03 SA   (Summarizer)        섹션별 "개념 위주 산문" 요약 (수치·세팅 버림)
 ▼ 04 EEA  (Entity Extraction) 개념 + 정의(다중) 추출
 ▼ 05 align (정규화)            개념이 기존 맵에 있나/새것인가 판정 (merge/new)
 │
 ▼ (예정) 06 REA → 07 SAA → 08 CRA → 09 통합
```

- 파일: `agents/agent_01_ingestion.py` ~ `agent_05_alignment.py`
- 프롬프트: `agents/prompts/` (영어 본문 + 한글 1:1 번역 주석. import 시 영어만 따라옴)
- 산출물: `data/parsed/{arxiv_id}/{arxiv_id}_0N.*` (논문별 폴더 + 단계 번호)

## 2. 단계별 정책

### 01 IA — 파싱·청소

- **역할:** PDF → 깨끗한 본문 마크다운 + 메타데이터(arxiv_id).
- **결정:**
  - 파싱은 `pymupdf4llm.to_markdown(use_ocr=False)` (검증된 라이브러리 원칙, 직접 안 짬).
  - `## REFERENCES` 헤더 이후 통째 삭제 → references + appendix 동시 제거.
  - 표(`|`로 시작하는 행)·그림 잔해(`picture text` 블록, `intentionally omitted`)·페이지 가구(4회+ 반복되는 짧은 줄 = 러닝 헤더, 페이지번호만 있는 줄) 규칙 제거.
- **근거:**
  - 표·그림이 깨지는 건 버그가 아니라 **무료 필터** — 거기 담긴 건 수치·시각자료라 개념맵에 불필요. 비전 모델로 표 살리는 건 인프라 과잉투자.
  - references 컷은 의미 판단이 필요 없는 규칙 작업 → IA에서 싸게. (KARMA는 RA 관련성 컷에 맡기지만, 우린 그 컷을 보류했으므로 IA로 당김.)
  - **함정:** "appendix" 단어가 본문에 흩어져 있어 단어 매칭 금지 → 헤더 줄(`## REFERENCES`)만 경계로.
- **KARMA 대비:** KARMA IA는 LLM으로 OCR 오류 정규화(생의학 PDF가 지저분해서). 우리 arxiv PDF는 깨끗해서 규칙 변환으로 충분 → 사실상 "전처리". 입력이 거칠어지면(코드베이스 등) LLM 정규화로 진화 가능.
- **주의:** 청소 규칙은 pymupdf4llm 출력 모양에 **종속**. 파서 바꾸면 재작성. 새 논문은 `inspect_pdf.py`로 헤더 모양 먼저 점검.

### 02 RA — 섹션 분할

- **역할:** 본문을 `## 헤더`마다 세그먼트로 분할 (ReAct는 14개).
- **결정:** **분할만 한다. 관련성 컷(δ)은 보류.**
- **근거:**
  - KARMA RA는 분할 + 관련성 점수(세그먼트가 그래프에 유의미한가) 컷. 우리는 컷을 **Collector(자동 수집) 단계로 미룸** — 컷 설계는 "입력이 얼마나 잡음 많나"에 달렸는데, 그건 수집기가 정해져야 알 수 있음. 지금 정하면 인프라 과잉투자.
  - 분할은 LLM 안 씀 — pymupdf4llm이 `## 헤더`를 잘 보존해서 규칙으로 충분. (KARMA는 LLM 분할이지만 그건 PubMed PDF가 헤더 구조를 안 남겨 의미로 복원해야 했기 때문.)
- **함의:** RA가 지금 "JSON 변환"처럼 가벼운 건 ① 관련성 컷을 뺐고 ② 헤더가 깨끗해 LLM 불필요하기 때문. 누락이 아니라 두 결정의 결과.

### 03 SA — 요약

- **역할:** 각 섹션을 "개념과 그 연결이 드러나는 짧은 산문"으로 압축. (파이프라인에서 처음 LLM 등장.)
- **결정:**
  - **수치·실험세팅·하이퍼파라미터 버림** (EM 35.1, PaLM-540B, 32-shot 등). 개념의 정체성에 필요한 수치만 남김.
  - **외부지식 절대 금지** — 입력에 있는 것만. LLM 사전지식으로 채우면 할루.
  - **산문으로** (불릿·트리·계층 목록 금지). 개념을 문장으로 *연결*해 서술 (관계 판정은 안 함, 그건 다음 단계).
  - 길이 고정 안 함 — 개념 밀도가 정함. 분량 맞추려 개념 빼기 금지.
- **근거:**
  - 맵이 "지형도"라 수치는 노이즈. (KARMA SA는 정반대로 수치 verbatim 보존 — KARMA 맵은 수치가 곧 지식이라서. **목적이 프롬프트를 정반대로 만든 대표 사례.**)
  - KARMA "Summarizer 빼면 정확도 22.9%↓"는 요약 단계의 가치지, Reader/Summarizer 분리의 근거가 아님. 우리는 한 콜로 통합(Reader 역할이 이미 RA·evaluator에서 소진됨).
- **삽질 기록:** 처음 "concept map"이라 했더니 LLM이 시각적 트리를 그림. "concise prose" + 좋은 예시 하나로 교정. **프롬프트 단어 하나가 출력 형식을 좌우** — 예시가 금지 나열보다 강력.

### 04 EEA — 개념 추출

- **역할:** 요약문에서 개념 + 정의 추출. 출력: `{name, definitions: [...]}` 리스트.
- **결정:**
  - **개념 = 고유 이름 가진 것** (방법·기법·메커니즘·벤치마크). 속성구·막연한 지칭·일반 명사구·모델 인스턴스(PaLM, GPT-3)는 제외. 판별 테스트: "서베이 색인에 이름 항목으로 실릴 만한가?"
  - 정의는 **개념 자체**가 뭔지 (이 논문에서의 적용·대조 말고). 외부지식 금지.
  - **정의를 여러 개 보관** (다중 정의). 같은 이름은 묶되 정의는 안 버리고 리스트로.
  - **기존 맵은 안 봄** (그건 align). EEA는 "이 논문에 이런 개념들이 있다"만.
- **근거 — 폐기한 것들 (중요):**
  - **status 플래그 폐기.** "이 개념을 논문이 정의했나(defined)/사용만 했나(used)"를 LLM이 판정하게 했으나 **불안정 + 과부하.** (LLM에 추출+정의+메타판정 3중.) 분류는 align의 측정(임베딩 거리)에 맡기는 게 정직. → "이름값이 아닌 적합성으로" 원칙.
  - **빈 문자열/필드 유무로 상태 표현 금지** — 부재로 의미 싣기는 버그와 정상을 구분 못 함.
  - **aliases 폐기** — 약어 병합 보조용이었으나, ① 이름 비교를 안 하기로 했고 ② 소비처가 없어 안 씀. (필요해지면 그때.)
  - **"긴 정의 택하기" 폐기** — 같은 개념도 섹션마다 정의 품질이 다름(ReAct 12개 출현: 깨끗한 것~GPT-3 코드 오염까지). 길이 ≠ 품질, 오히려 맥락 붙으면 길어짐(=오염). 좋은 정의가 삭제될 위험 → **안 버리고 다 모아 align이 거리로 best 선택.**
- **dedup(group) 현재 역할:** 같은 이름(대소문자) 묶어 정의 리스트로 모으기만. 판단(같은 개념인가, 어느 정의가 best인가)은 align으로.

### 05 align — 정규화 (핵심 단계)

- **역할:** 각 개념이 기존 맵에 있나(merge)/새것인가(new) 판정. **판정만, 맵엔 안 씀** (실제 통합은 SAA·CRA·EA 후).
- **결정 — 2단계 (임베딩 후보 + LLM 확정):**
  1. **임베딩으로 후보 추림** (싼 거친 필터): 개념의 모든 정의 × 시드맵 노드 정의문 코사인 유사도. 상위 K(=2) 중 floor(=0.40) 넘는 것만 후보. 무관한 건(FEVER 0.22) 즉시 new.
  2. **LLM으로 확정** (정밀): 후보만 "이 둘 같은 개념?" 판정 (SAME/DIFFERENT).
- **LLM 판정 기준:**
  - SAME = "한 가지를 가리키는 두 이름" (약어 CoT=chain-of-thought, 동일 의미).
  - DIFFERENT = **변형·확장·발전형** (zero-shot CoT vs CoT), 특정 방법 vs 일반 개념 (STaR vs reasoning), 단순 관련/결합.
- **근거:**
  - **정의문으로 비교 (이름 X).** KARMA는 이름 임베딩(생의학은 도메인 임베딩 모델 SapBERT 등으로 이름만으로 식별 가능). AI 개념은 그런 모델이 없고 이름만으론 의미가 안 잡혀 → 정의문 임베딩. **이 선택이 "정의 어디서 오나/품질" 문제를 새로 만든, 우리만의 영역.**
  - **임베딩만으론 부족** — 같은 개념도 정의 표현 차이로 0.6대, 무관한 것도 0.58 → 점수가 안 갈려 단일 임계값 불가. 그래서 임베딩=거친 필터, LLM=의미 판정으로 역할 분담. (KARMA의 임베딩 후보 + LLM verification 다단계 정신과 일치.)
  - 이게 **핵심 기술 부채 ①("이름만 겹치면 억지연결")의 해소.** LLM이 점수가 아니라 의미로 판단 → 점수 낮은 진짜(CoT 0.417)는 살리고, 점수 높은 가짜(STaR 0.583→reasoning)는 거름.
- **검증:** 시드맵 9노드에 merge/new/boundary 의도를 심어 채점. CoT 계열만 merge, 변형·일반어 끌림·무관은 전부 new로 정확히 갈림.

## 3. 관통하는 원칙

1. **맵은 지형도, 디테일 저장소 아님** — 수치·세팅 버림. 디테일은 사용자가 나중에 파고듦.
2. **외부지식 절대 금지** — 모든 정의는 논문 근거만. LLM 사전지식 채우기 금지(할루).
3. **정의문으로 비교 (KARMA는 이름)** — AI 개념엔 도메인 임베딩 모델이 없어서. 이 선택이 우리만의 문제·차별점을 만듦.
4. **거친 한 바퀴 먼저** — 인프라 과잉투자 경계. 관련성 컷·파서 추상화·인자화는 실제 블로커 생길 때.
5. **이름값 아닌 적합성으로 도구 선택** — status(LLM 자기판정) 폐기, BKT 폐기와 같은 결.
6. **정보 안 버리고 판단 가능한 곳에서 결정** — 정의 다중 보관 → align이 best 선택.
7. **검증된 라이브러리 직접 안 짬** — pymupdf4llm 등.
8. **enrichment** — 빈약/오염된 정의는 버그가 아니라 "아직 본진 논문 안 옴". 논문 쌓이며 채워짐.

## 4. KARMA와 다른 점 (차별점 = 면접 서사)

| 항목 | KARMA | 우리 |
|---|---|---|
| 목적 | 정확한 분야 그래프 완성 | 내 커버리지 지형도 + self-model + 추천 |
| 비교 기준 | 엔티티 **이름** 임베딩 | 개념 **정의문** 임베딩 |
| SA 수치 | verbatim 보존 (수치가 지식) | 제거 (수치는 노이즈) |
| 입력 코퍼스 | 큐레이션된 PubMed 1,200편 | arxiv (넓음 → 관련성 컷 필요) |
| 정의 출처 문제 | 없음 (이름 기반) | 있음 (정의문 기반) → 다중 보관으로 대응 |
| 관련성 컷 위치 | RA (요약 전, 비용 절약) | 보류 → 검토 시 SA 후가 정확도 유리 |

## 5. 미결 / 다음 과제

- **06 REA (관계 추출):** 노드 쌍 사이 `is_a` / `part_of` / `depends_on` 긋기. 점 다 찍었으니 선. 입력은 요약문 + 노드 리스트, 같은 요약문에 함께 나온 개념끼리. merge 노드는 끝점을 시드노드로, new는 신규 노드로.
- **07 SAA (스키마 정합):** 관계 타입을 맵 전체 스키마와 정합. **REA와 별개 단계**(맵 전체와 비교 = 글로벌 정합 자리). 단 관계 3종 화이트리스트라 "새 타입 분류"가 "3종 매핑 or 폐기"로 가벼움.
- **08 CRA (충돌):** 후순위.
- **09 통합:** 위 다 거친 뒤 실제 맵에 쓰기. **지금 align은 판정만, 맵 안 씀.**
- **정의문 점수 낮게 깔림:** 같은 개념도 0.6대(정의 한 문장이라). 현재 LLM 판정으로 우회. 정의 풍부화는 할루 위험으로 기각.
- **변형의 관계 처리:** zero-shot CoT 등을 별개 노드로 두기로 → 나중에 `depends_on`/`is_a`로 CoT와 이어야.
- **두 번째 논문 (CoT 논문 등):** 넣으면 enrichment 검증 — 이번에 new로 떨어진 변형·일반어가 본진 정의로 채워지는지.
- **관련성 컷 + 파서 추상화 + PAPER 인자화:** Collector / 두 번째 입력 생길 때.
